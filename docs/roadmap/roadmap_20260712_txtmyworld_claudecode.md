## 2. 버전별 종료 기준 (Definition of Done)

### v0.1 MVP
- 세 소스 페어링·통합 조회·스키마 검증·폴백 격리.
- 로컬 bge-m3 임베딩 + sqlite-vec KNN + 교차소스 의미 매칭 + 기본 의미 클러스터 + 의미 기반 질의.
- 기간·빈도 융합 스코어(기본 가중치), 추천 피드 + 탐색.
- 주제 카드 CRUD + 근거 표시, 리스트 뷰 동등성, 키보드·고대비.
- X1 스키마(v1.1) **정의** + 소비자 스캐폴딩(있으면 사용, 없으면 폴백).
- X2 동의형 환류 **기본 흐름**(미연결 폴백).
- 인터넷·클라우드 없이 전 기능 동작.

### v0.5
- 소스측 벡터 공유(A) 최소 1개 소스 실연동, 공간 정합/재임베딩 검증.
- 의미 클러스터 품질 개선(HDBSCAN), 드리프트 타임라인.
- HNSW 승격 경로 확보.

### v1.0 RC
- **X1:** 준비된 소스 벡터 공유 정착, 미지원 혼재에도 통합 발견 정상. (통합 스펙 §2.7)
- **X2:** 동의→쓰기→이력→멱등 update E2E, 본문 미포함 검증. (통합 스펙 §3.7)
- 3축 융합 스코어 최종 튜닝, 다중 홉 의미 추론.
- 성능·접근성·프라이버시 기준 충족(PRD §7·§10·§14).
- 릴리즈 패키지(README·프라이버시 노트·알려진 한계).

## 4. 리스크·완충

- **패밀리 의존성 지연:** TXTMyWorld는 소스 하나(Diary)만으로도 착수·검증 가능하게 설계 → 일정 격리.
- **벡터 규모 폭증:** sqlite-vec→HNSW 승격 경로 사전 확보, 상한·저사양 모드.
- **범위 확산(시각화 비대):** "생성"이 핵심. 시각화는 발견 보조로 고정.
