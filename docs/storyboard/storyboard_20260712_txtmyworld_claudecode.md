## S0. 온보딩 · 소스 페어링

```
┌───────────────────────────────────────────┐
│  TXTMyWorld  v0.1.0                        │
│  "내 맥락들이 만나는 곳"                    │
│                                           │
│  소스 연결                                 │
│  [ TXTDiary   ● 실행중  ]  [연결]          │
│  [ TXTBrain   ○ 꺼짐    ]  [나중에]        │
│  [ TXTAIMemory● 실행중  ]  [연결]          │
│                                           │
│  로컬 엔진: bge-m3 준비됨 ✓                 │
│  벡터 공유(X1): TXTDiary 지원 · Brain 미지원 │
│                             [시작하기]     │
└───────────────────────────────────────────┘
```
- 각 소스 승인 다이얼로그(각 앱)에서 사용자가 동의 → 페어링 토큰 저장.
- 미실행 소스는 "나중에"로 건너뛰고 이후 설정에서 연결. Diary 하나만으로도 시작 가능.
- 접근성: 상태는 색+텍스트 라벨(실행중/꺼짐/지원/미지원) 병행.

## S1. 홈 — 추천 피드

```
┌───────────────────────────────────────────┐
│  발견  |  탐색  |  보관함        [설정]      │
│───────────────────────────────────────────│
│  이번 주 새로 이어진 것 (3)                 │
│                                           │
│  ◇ [클러스터]  관측 문제: 세 갈래           │
│     관측자(일기)·측정문제(문서)·observer... │
│     의미유사 0.82 · 최근 급상승   [열기]    │
│                                           │
│  ◇ [갭]  '회복탄력성' — 문서엔 있고 기록엔 X │
│     지식은 있는데 경험 기록이 비어 있음      │
│                                   [열기]    │
│                                           │
│  ◇ [드리프트]  '몰입'의 감정 이동           │
│     3개월간 중립→긍정으로                    │
│                                   [열기]    │
└───────────────────────────────────────────┘
```
- 발견 유형은 대괄호 텍스트 배지(색+라벨). 근거 수치는 텍스트 병기.
- 자동 계산(하이브리드의 추천 축). 비어 있으면 "탐색으로 직접 파보기" 안내.

## S2. 발견 상세 — 근거

```
┌───────────────────────────────────────────┐
│  ← 발견   [클러스터] 관측 문제: 세 갈래      │
│───────────────────────────────────────────│
│  구성 키워드                                │
│   • 관측자        일기(txtdiary)  12회      │
│   • 측정 문제     문서(txtbrain)   8회      │
│   • observer eff. AI대화(aimemory) 5회      │
│                                           │
│  근거                                      │
│   의미 유사도  0.82 (bge-m3)               │
│   기간 겹침    2026-05 ~ 07 (겹침 강)      │
│   빈도 신호    상승                         │
│   신뢰도       강한 신호                    │
│                                           │
│  [ 주제 카드로 만들기 ]   [ 관심 없음 ]     │
└───────────────────────────────────────────┘
```
- 리스트/문장 동등: 스크린리더가 "관측자는 일기에서 12회… 세 소스가 의미유사 0.82로 묶임"을 완전한 문장으로 읽음.
- 각 키워드 → 소스 딥링크(원본 앱 열기).

## S3. 주제 카드 생성

```
┌───────────────────────────────────────────┐
│  주제 카드 만들기                           │
│  이름  [ 관측 문제: 나의 세 갈래        ]   │
│        제안: "관측과 측정" / "관찰자 효과"   │
│  메모  [ ...                            ]   │
│  구성  관측자 · 측정 문제 · observer effect │
│  근거  스냅샷 저장됨(0.82 / 05~07 / 상승)   │
│  □ 이 주제를 AI 기억에 남기기 (환류)        │
│                         [ 저장 ]           │
└───────────────────────────────────────────┘
```
- AI 라벨 제안: 로컬 기본, 동의 시 DeepSeek 고품질(전송 항목 고지).
- 환류 체크 시 S3-확인 다이얼로그(전송 항목: 제목·키워드·근거요약·딥링크, 본문 없음) → X2 MCP 쓰기.

## S4. 탐색 캔버스

```
┌───────────────────────────────────────────┐
│  탐색                                      │
│  소스 [✓일기 ✓문서 ✓AI]  기간 [최근 30일]  │
│  축   [✓벡터 ✓빈도 ✓기간]  유사도 컷 [0.6] │
│  의미 질의 [ "이런 느낌의 주제" ____ ] [찾기]│
│───────────────────────────────────────────│
│  결과 (관계 뷰 / 리스트 뷰 전환)            │
│   ○──○  노드=키워드(크기=빈도)             │
│    \ /   엣지=의미유사·동시출현             │
│     ○    ↳ 클릭 시 근거·딥링크              │
└───────────────────────────────────────────┘
```
- 관계 뷰 ↔ 리스트 뷰 토글, 항상 기능 동등. 키보드 노드 탐색.
- 의미 질의: 자연어/예시 → 임베딩 → 최근접 이웃.

## S5. 주제 카드 보관함

```
┌───────────────────────────────────────────┐
│  보관함                            [내보내기]│
│  • 관측 문제: 세 갈래   확정  환류됨 ✓       │
│  • 회복탄력성 갭        초안  환류 안 함     │
│  • 몰입의 이동          보관                 │
│   ↳ 편집 · 환류 토글 · 삭제(soft)           │
└───────────────────────────────────────────┘
```
- 환류 상태는 텍스트("환류됨/안 함")+아이콘. 카드 수정 시 update 재환류(재확인).
