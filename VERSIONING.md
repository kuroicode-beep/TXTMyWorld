# 버전 규칙 (Semantic Versioning)

TXTMyWorld는 표준 `MAJOR.MINOR.PATCH` 세미버를 따르며 `0.1.0`에서 시작한다.

| 단계 | 규칙 | 예시 |
|-----|-----|-----|
| 첫 버전 | 초기값 | `v0.1.0` |
| 마이너 업데이트 (버그픽스/소기능) | PATCH +1 | `v0.1.1`, `v0.1.2` … |
| 주요 업데이트 (기능/UI 마일스톤) | MINOR +1, PATCH 리셋 | `v0.2.0`, `v0.3.0` … |
| RC 완료 (첫 안정판) | | `v1.0.0` |
| 대규모 업데이트 (호환 깨짐/데이터 구조 변경) | MAJOR +1 | `v2.0.0`, `v3.0.0` … |

- MAJOR: 호환 깨짐 / 데이터 구조 변경
- MINOR: 기능 추가 / UI 개편
- PATCH: 버그 수정

## 버전 관리 방식

- 루트 `VERSION` 파일이 현재 버전의 단일 소스.
- 코드에서는 `APP_VERSION` 상수 + `VERSION_HISTORY` 리스트(버전, 날짜, 요약)로 관리하고, 기능 추가 시마다 갱신한다.
- 버전 표시: 로고 옆(로고 없으면 창 제목 + 화면 상단 모서리)에 `vX.Y.Z` 상시 표시.

## 히스토리 메뉴 요구사항 (UI 앱 필수)

이 앱이 UI를 갖게 되면, 설정(또는 정보) 화면 안에 **"업데이트 히스토리 / 업데이트 내역"** 메뉴를 두고 버전별 날짜·변경 요약(최신순, 버전당 2~4줄)을 사용자에게 보여준다. CHANGELOG.md 같은 git 로그성 기록과는 별개로, 앱 안에서 바로 확인 가능해야 한다.
